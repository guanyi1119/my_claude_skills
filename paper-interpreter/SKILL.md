# 论文解读 Skill

## 概述

这是一个用于深度解读学术论文的skill，能够帮助用户快速理解和分析研究论文的核心内容，并支持论文归档管理。

## 功能特性

- 📄 支持PDF和纯文本格式论文
- 🌐 中英文论文自动识别和处理
- 📊 结构化提取关键信息
- 💡 专业术语解释
- 🎯 重点突出主要贡献点
- 🔬 详细分析方法论和消融实验
- 📚 论文归档功能：
  - 可选择归档或不归档
  - 按标签分类保存
  - Excel文件管理，支持超链接
  - 自动重命名和文件管理

## 工作流程

### 第一步：确定输入方式

首先，确认用户提供的论文形式：
- 如果是PDF文件，优先检测是否存在其他PDF处理skill
- 如果是直接粘贴的文本，直接使用
- 如果是文件路径，先读取文件

**PDF处理：**
1. **检测可用的PDF skill：**
   - 首先检查是否存在 `document-skills:pdf` 或其他能够下载和提取PDF的skill
   - 如果存在，优先使用该skill来提取PDF文本内容
   - 如果不存在，则使用内置的PDF提取脚本

2. **使用内置PDF提取脚本（当无其他PDF skill时）：**
```python
# 使用scripts/pdf_extractor.py提取文本
python scripts/pdf_extractor.py <pdf文件路径>
```

### 第二步：论文语言检测

自动检测论文语言（中文或英文），并据此调整分析策略：
- 英文论文：重点关注Abstract、Introduction、Methodology、Experiments、Conclusion等章节
- 中文论文：重点关注摘要、引言、方法、实验、结论等章节

### 第三步：结构化分析

按照以下框架进行深度分析：

#### 1. 论文概览
- 标题
- 作者信息（如可获得）
- 发表年份/会议/期刊（如可获得）
- 一句话总结

#### 2. 研究背景与问题
- 研究领域背景
- 现有工作的局限性
- 本文要解决的核心问题

#### 3. 主要贡献点（重点）
**以列表形式清晰列出3-5个主要贡献：**
```
贡献点1：[贡献名称]
- 具体描述：详细说明这个贡献是什么
- 创新点：与之前工作的区别
- 重要性：为什么这个贡献很重要

贡献点2：[贡献名称]
...
```

#### 4. 方法论细节
对于每个主要贡献，详细描述：
- 核心算法/模型架构
- 关键技术步骤
- 数学公式（如适用，用LaTeX格式）
- 实现细节和技巧

#### 5. 实验设置
- 数据集（名称、规模、来源）
- 评估指标
- 基线方法对比
- 实验环境（硬件、软件配置）

#### 6. 消融实验分析（如有）
**详细分析每个消融实验：**
```
消融实验1：[实验名称]
- 实验目的：验证什么假设
- 实验设计：移除/修改了哪个组件
- 实验结果：具体数值变化
- 结论：验证了什么结论

消融实验2：[实验名称]
...
```

#### 7. 主要结果
- 关键实验数据表格（简洁摘要）
- 与基线方法的对比
- 结果分析和解释

#### 8. 结论与未来工作
- 本文总结
- 主要局限性
- 未来研究方向

#### 9. 关键术语解释
解释论文中出现的3-5个重要专业术语：
```
术语1：[术语名称]
- 定义：简明解释
- 在本文中的含义：如何应用
```

## 关键原则

1. **准确性优先**：确保提取的信息准确无误，不确定的内容明确标注
2. **重点突出**：主要贡献点和方法论是核心，要详细展开
3. **结构化呈现**：使用可展开的details标签让输出清晰易读
4. **交互友好**：结尾提供引导性问题，鼓励进一步交流
5. **语言适配**：根据论文语言选择合适的分析和表达方式

### 第五步：保存解读报告并询问归档

#### 5.1 保存解读报告
在解析完成后，先保存论文解读报告到 `interpretations/` 目录下：

#### 5.2 归档询问
```
## 📚 归档询问

**是否需要归档这篇论文？**
[用户可以选择：归档 / 不归档]
```

#### 如果用户回答"不归档"
- 删除临时生成的PDF文件和提取的文本文件
- 删除已保存的MD文件
- 结束流程

```bash
# 删除临时文件
rm -f arxiv_paper.pdf extracted_paper.txt
rm -f interpretations/{arxiv_id}_interpretation.md
```

#### 如果用户回答"归档"
**第一步：询问归档原因**
```
## 📚 归档原因

**请输入归档这篇论文的原因：**
[用户输入归档原因]
```

**第二步：询问归档标签**
```
## 📚 归档标签

**请输入这篇论文的归档标签（用于分类）：**
[用户输入标签，如"计算机视觉"、"自然语言处理"等]
```

**第三步：执行归档**
使用 `scripts/archiver.py` 脚本来处理归档，重命名PDF文件和MD文件为论文标题格式：

```python
# 归档调用示例
python scripts/archiver.py <base_dir> --title "论文标题" --year "2024" --affiliation "作者单位" --venue "CVPR" --keywords "3D重建,计算机视觉" --tag "计算机视觉" --reason "研究兴趣相关" --pdf arxiv_paper.pdf --md interpretations/arxiv_2603.25053.md
```

### 归档管理

#### Excel文件结构
归档信息保存在 `archive/paper_archive.xlsx` 中：

| 列标题 | 说明 | 示例 |
|-------|------|------|
| 标题 | 论文标题 | GaussFusion: Improving 3D Reconstruction in the Wild... |
| 年份 | 发表年份 | 2026 |
| 作者单位 | 作者所属机构 | Stanford University |
| 收录会议/杂志 | CVPR、ICML、IEEE等 | CVPR 2026 |
| 关键词 | 关键词（逗号分隔） | 3D高斯splatting,视频生成, artifact去除 |
| 归档原因 | 用户输入的归档原因 | 研究兴趣相关，方法有创新 |
| 归档日期 | 归档时间 | 2026-04-02 11:52:33 |
| PDF路径 | 本地PDF文件（超链接） | file:///d:/Workspace/PaperReader/archive/pdf/... |
| 解读报告路径 | 本地MD文件（超链接） | file:///d:/Workspace/PaperReader/archive/md/... |

#### 文件命名规则
- PDF文件：`{论文标题}_{年份}.pdf`（清理非法字符）
- MD文件：`{论文标题}_{年份}_解读.md`

#### 目录结构
```
paper-interpreter/
└── archive/
    ├── paper_archive.xlsx  # 归档Excel文件
    ├── pdf/                # 论文PDF文件
    └── md/                 # 论文解读报告
```
